## 3. **세계관 및 캐릭터 설정**

### 3.1 세계관

#### ▸ **현대 경복궁 → 조선 경복궁 타임리프 구조**

- 현대 경복궁 야간 개장은 현실적 묘사
- 빛나는 전통 보따리: '시간문(문을 여는 트리거)'
- 조선 경복궁은 살짝 판타지화: 단청이 살아 움직이고, 길·광장에 과장된 공간감
- '시간 시험'은 전통놀이를 변형한 도깨비의 시험이라는 설정

### 3.2 주요 캐릭터

#### 👹 **수문도깨비(Guardian Goblin)**

**배경**
- 경복궁 시간을 지키는 수문장 역할을 하던 도깨비
- 전통이 사라져가는 것에 대한 슬픔과 경계심이 강함

**성격**
- 장난기 있으나 서늘한 분위기
- 플레이어를 시험하지만 파괴적이지 않음
- 마지막엔 따뜻한 진심 전달

**동기**
- "전통이 잊히는 것을 막고, 다시 현대에 놀이를 피워내게 하려는 목적"

**비주얼(컨셉 가이드)**
- 사람 크기보다 약간 큰 실루엣
- 뿔·탈 형상
- 등장 시 단청 파편 조각들이 빛나며 모여 형성됨

## **Area_02: ROUND 1 - 판돌 건너기 (유리발판 전통화 버전)**

### 1) **레벨 구조**

- 공중에 길게 이어진 돌다리
- 두 줄 발판 (좌/우 선택)
- 한 발판은 "안전", 하나는 "깨짐(낙하)"
- 총 10~12칸

### 2) **규칙 / 시스템**

| 항목 | 설명 |
|------|------|
| 발판 종류 | 2종: 고정 발판 (Collider On) / 깨지는 발판 (Collider Off + 파괴 이펙트) |
| 랜덤 배치 | 각 단계마다 좌/우 중 하나가 랜덤하게 안전 발판으로 지정 |
| 밟았을 때 | 안전: 유지 / 깨짐: 아래로 추락 |
| 낙하 시 | 시작 지점으로 자동 리스폰 |
| 승리 조건 | 마지막 발판 너머 골 플랫폼 도달 |

### 3) **도깨비 멘트**

"첫 번째 시험, 눈에 보이는 길만이 정답은 아니다.

직감을 믿어 앞으로 나아가라."

### 4) **구현 상태**

## 7. **기술적 요구사항 및 제약조건**

### ▸ 개발 환경

| 항목 | 사양 |
|------|------|
| 엔진 | Unity 2022.3 LTS |
| SDK | TwentyOz VIVEN SDK |
| 스크립팅 | Lua (xLua 기반) |
| VR 방식 | 6DoF Teleport 방식 |

### ▸ 필수 구현 요소

- VR 이동 (텔레포트 방식)
- 영역 전환 (페이드 + 텔레포트)
- 낙하 + 리스폰 로직
- 물리 투척 (VivenGrabbableModule + Rigidbody)
- 단일 씬 내 5개 영역 관리

### ▸ 최적화 제약

- 오브젝트 수 최소화
- 비활성 영역 SetActive(false) 처리
- 발판 파괴는 Collider Off + 메쉬 숨김 방식 (실제 파티클 대신)
- 비석 물리 Simplicity → 충돌 기반 기울기 검사

### ▸ VIVEN SDK 제약 사항

- `Debug.Log`만 사용 (LogWarning/LogError 사용 불가)
- Transform 직접 주입 불가 → GameObject 주입 후 `.transform` 접근
- TMP_Text 직접 주입 불가 → `GetComponentInChildren(typeof(TMP_Text))` 사용
